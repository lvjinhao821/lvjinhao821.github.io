创建函数的两种方式

定义函数的方式有两种：

  • 函数声明

    函数声明在js解释器刚开始执行时就会先把函数进行提升，故sayHi()调用放在函数定义前也可以执行成功

    sayHi();
function sayHi(){
    alert("hi!")
}

  • 函数表达式

    什么是函数表达式？表达式是有值的代码段，函数对象也是一个值。因此可以以变量声明的方式把一个匿名函数赋值给一个变量。（没有名字的函数叫匿名函数），使用前必须先赋值。

    sayHi();     //错误：函数还不存在
var sayHi = function(){
    alert("hi!");
}

    变量sayHi的值就是指向函数对象的指针，因为是引用数据类型（复杂数据类型），所以变量sayHi的值是一个存放在栈中指向堆中函数对象所在位置的指针。从本质上来讲，和使用函数声明的方式定义一个有名字的函数是一样的。

递归

递归函数是在一个函数通过名字调用自身的情况下构成的

//通过函数名字调用
function factorial(num){
    if (num <= 1){
        return 1
    } else {
        return num * factorial(num - 1);
    }
}

arguments.callee 是一个指向正在执行的函数的指针，可以用它来实现递归的调用。更加保险。但严格模式访问这个属性会导致错误。

//通过arguments.callee调用
function factorial(num){
    if (num <= 1){
        return 1
    } else {
        return num * arguments.callee(num - 1);
    }
}

通过命名函数的表达式，将函数的名字用来调用也可以。

//factorial函数内外都可使用，f用于函数内
var factorial = function f(num){
        if (num <= 1){
            return 1
        } else {
            return num * f(num - 1);
        }
    }

闭包

①闭包是指有权访问另一个函数作用域中的变量的函数（另一个函数，通常指的是包含闭包函数的外部函数）。创建闭包最常见的方式，就是在一个函数内部创建另一个函数。(并没有说，内部创建的函数，必须要return返回出来！！！)

②深入理解作用域链；

  • 当某个函数被调用时，会创建一个作用域及相应的作用域链。

  • 然后，使用arguments和其他命名参数的值来初始化函数的活动对象。

  • 在作用域链中，外部函数的活动对象始终处于第二位，外部函数的外部函数的活动对象处于第三位……直至作用域链终点的全局执行环境。

③作用域链本质上是一个指向变量对象的指针列表，它只引用但不实际包含变量对象。

④在函数执行过程中，为读取和写入变量的值，就需要在作用域链中查找变量；

  • 通过作用域链“列表”，子对象会一级一级地向上寻找所有父对象的变量。所以，父对象的所有变量，对子对象都是可见的，反之则不成立。

⑤由作用域链中查找变量的过程，我们知道不能从函数外面访问函数内的变量。那么我们就可以在函数内部，创建一个子函数，利用子函数能访问父函数的特点，间接的访问函数内部的变量。这就解释了创建闭包的常见方式是在一个函数内部创建另一个函数。

⑥闭包可以用在许多地方，它最大用处有两个，一个是前面提到的可以读取函数内部的变量，另一个就是可以让这些变量的值始终保存在内存中。

一般来说，当函数执行完毕后，局部活动对象就会被销毁，内存中仅保存全局作用域(全局作用域的变量对象)，但是闭包的情况有所不同，原因是：在函数A中定义一个新函数，新函数作为返回值赋给了一个全局变量，这就导致了新函数始终在内存中，而新函数的存在依赖于函数A，因此函数A也始终在内存中，不会再调用结束后，被垃圾回收机制回收。

闭包会引用包含函数的整个活动对象，即使有些变量闭包没有直接引用，但函数的活动对象中也仍然会保存一个引用，因此有必要将没有引用的变量设置为null，减少其引用数，收回其占用的内存。

⑦注意！由于闭包所保存的是整个变量对象，因此闭包只能取得包含函数中任何变量的最后一个值。可看下面的例子；

function createFunctions(){
    var result = new Array();
    for (var i = 0; i < 10; i++){
        result[i] = function(){
            return i;
        };
    }
    return result;
}

var funcs = createArray();
for (var i = 0; i < funcs.length; i++) {
     document.write(funcs[i]() + "<br />");  //调用闭包函数，获取i值
}
//打开html文件，显示的每项都是10

这个函数会返回一个函数数组，每个函数都会返回10。因为每个函数的作用域链中都保存着createFunctions()函数的活动对象，变量值i的获取是在闭包函数调用的时候才从外部函数的变量对象中获取。而这个时候，外部函数早就完成for循环使i=10了。

我们可以通过创建另一个匿名函数强制让闭包的行为符合预期

function createFunctions(){
    var result = new Array();
    for (var i = 0; i < 10; i++){
        result[i] = function(num){
            return function(){
                return num;
            };
        }(i);
    }
    return result;
}

这个版本，没有直接把闭包赋值给数组，而是定义了一个匿名函数，并将立即执行该匿名函数的结果赋给数组。这里的匿名函数有一个参数num，也就是最终的函数要返回的值。在调用每个匿名函数时，我们传入了变量i，由于函数的参数是按值传递的，所以就会将变量i的当前值复制给参数num。而在这个匿名函数的内部，又创建并返回了一个访问num的闭包。这样一来，result数组中的每个函数都有自己num变量的一个副本，因此就可以返回各自不同的数值了。（0 1 2 3 4 5 6…10）

⑧在闭包中使用this对象可能会导致些问题，因为this对象是在运行时基于函数的执行环境绑定的：在全局函数中，this等于window，而当函数作为某个对象的方法调用时，this等于那个对象。不过，匿名函数的执行环境具有全局性，因此其this对象通常指向window。

var name = "The window"
var object = {
    name: "my object",
    getName: function(){
        teturn function(){
            return this.name;
        };
    }
}
alert(object.getNameFunc()());   //"The window"(非严格模式下)

不过，把外部作用域中的this对象保存在闭包能访问到的变量里，就可以让闭包访问该对象了。

var name = "The window"
var object = {
    name: "my object",
    getName: function(){
        var that = this;
        teturn function(){
            return that.name;
        };
    }
}
alert(object.getNameFunc()());  //my object

模仿块级作用域

在有块级作用域的语言中，变量i只会在for循环的语句块中由定义，循环一旦结束，变量i就会被销毁。而在javascript语言中，由于其没有块级作用域，因此从它有定义开始，就可以在函数内部随处访问它。可以利用匿名函数自执行来模仿块级作用域，使得块级作用域中的变量，只能在块级作用域中访问。

function(){
    //这里是块级作用域
}();    //出错！！

(function(){
    //这里是块级作用域
})();

由于Javascript将function关键字当作一个函数的声明，而函数声明后面是不能跟圆括号的。因此要给他加上一对圆括号，将函数声明转换成函数表达式，就不会报错。

无论在什么地方，只要临时需要一些变量，就可以用私有作用域。也被常用在函数外部，从而限制向全局作用域中添加过多变量和函数

私有变量

任何在函数中定义的变量，都可以认为是私有变量。利用闭包，可以访问到这些变量。我们把有权访问私有变量和私有函数的公有方法称为特权方法，有两种在对象上创建特权方法的方式；

  • 在构造函数中定义特权方法

    function MyObject(){
    //私有变量和私有函数
    var privateVariable = 10;
    function privateFunction(){
        return false;
    }
    //特权方法
    this.publicMethod = function(){
        privateVariable++;
        return privateFunction(); 
    }
}

    在创建MyObject实例后，除了使用publicMethod()这一个途径外，没有任何方法可以直接访问privateVariable和privateFunction()。

    利用私有和特权成员，可以隐藏那些不应该被直接修改的数据，如；

    function Person(name){
    this.getName = function(){
        return name;
    };
    this.setName = function(value){
        name = value;
    };
}

var person = new Person("jinhao");
alert(person.getName());      //"jinhao"
person.setName("yimiao");
alert(person.getName());      //"yimiao"

    getName()和setName()都有权在构造函数外部使用，而且都有权访问私有变量name。但在Person构造函数外部，没有任何方法访问name。

    构造函数模式的缺点是针对每一个实例都会创建同样一组方法。

静态私有变量

  • 在私有作用域中定义私有变量或函数，同样也可创建特权方法

    (function (){
    //私有变量和私有函数
    var privateVariable = 10;
    function privateFunction(){
        return false;
    }

    //构造函数
    MyObject = function(){
    };

    //特权方法
    MyObject.prototype.publicMethod = function(){
        privateVariable++;
        return privateFunction(); 
    }
})();
var person = new MyObject("jinhao");
alert(person.publicMethod());      //"false"

    公有的特权方法是在原型上定义的。这个模式在定义构造函数时并没有使用函数声明，而是使用了函数表达式。函数声明只能创建局部函数，那并不是我们想要的。出于同样原因，使用函数表达式也没有使用关键字var。记住：初始化未经声明的变量，总是会创建一个全局变量，就能够在私有作用域之外被访问到。但也要知道，在严格模式下，未经声明的变量赋值会导致错误。

    以这种方式创建静态私有变量会因为使用原型而增进代码复用，但每个实例都没有自己的私有变量。

模块模式

前面的模式都是用于为自定义类型创建私有变量和特权方法的。模块模式则是为单例创建私有变量和方法，单例指的就是只有一个实例的对象，javascript是以对象字面量的方式来创建单例对象的。

———————————没懂———————————

var singleton = {
    name: value,
    method: function(){
          //这里是方法的代码
    }
}；

模块模式通过为单例添加私有变量和特权方法能够使其得到增强，语法形式如下；

var singleton = function(){
    //私有变量和私有函数
    var privateVariable = 10;
    function privateFunction(){
        return false;
    }
    return {
        publicProperty: true;
        publicMethod: function(){
            privateVariable++;
            teturn privateFunction();
        }
    };
}();

如果必须创建一个对象并以某些数据进行初始化，同时还要公开一些能够访问这些私有数据的方法，那么就可以使用模块模式。

var application = function(){
    var components = new Array()
    components.push(new BaseComponent())

    return {
        getComponentCount: function(){
            return components.length
        },
        registerComponent: function(component){
            if(typeof component === "object"){
                components.push(component)
            }
        }
    }
}()

增强的模块模式

增强的模块模式适合那些单例必须是某种类型的实例，同时还必须添加某些属性和方法对其加以增强的情况.

var application = function(){
    var components = new Array()
    components.push(new BaseComponent())
    // 单例必须是某种类型的实例
    var app = new ComponentList()

    app.getComponentCount = function(){
        return components.length
    }
    app.registerComponent = function(component){
        if(typeof component === "object"){
            components.push(component)
        }
    }
    return app
}()
