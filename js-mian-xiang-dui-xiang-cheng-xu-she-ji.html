理解对象

ECMAScript把对象定义为：“无序属性的集合，其属性可以包含基本值、对象或函数。”可以ECMAScript的对象想象成散列表，无非就是一组名值对，其中值可以是数据或函数。

每个对象都是基于一个引用类型创建的，可以是原生类型，也可以是自定义类型。

创建自定义对象

创建自定义对象的方式；

//创建一个Object实例，然后为其添加属性和方法
var person = new Object();
person.name = "jinhao";
person.age = 22;
person.sayName = function(){
    alert(this.name);
};
//通过对象字面量的方式
var person = {
    name: "jinhao",
    age: 22,
    sayName: function(){
        alert(this.name);
    }
};

数据属性

ECMAScript中有两种属性：数据属性和访问器属性。

数据属性 作用
[[Configurable]] 表示能否通过delete删除属性，能否修改属性的特性，能否把属性修改为访问器属性。对于直接在对象上定义的属性，该特性默认值为true。
[[Enumerable]] 表示能否通过for-in循环返回属性。对于直接在对象上定义的属性，该特性默认值为true。
[[Writable]] 表示能否修改属性的值。对于直接在对象上定义的属性，该特性默认值为true。
[[Value]] 表示这个属性的值。写入属性值时保存在这里，读取属性值时在这读取，该特性默认值为undefined

用ES5的Object.defineProperty()方法，可修改属性的默认特性。接收三个参数；

  • 属性所在对象
  • 要修改的属性的名字(要加引号“xxx”)
  • 一个描述对象
    • configurable、enumerable、writable和value中的一个或多个。
var person = {};
Object.defineProperty(person, "name", {
    configurable: false,
    value: "jinhao"
});

alert(person.name);      //"jinhao"
person.name = "yimiao";  //不能修改属性特性，因此[[value]]值不会在变了
alert(person.name);      //"jinhao"

用该方法，新增一个属性时，configurable、enumerable、writable默认值都为false。修改已存在的属性时，没有此限制。

大多数情况下是没有必要用该方法提供的高级功能的。

访问属性

访问属性 作用
[[Configurable]] 表示能否通过delete删除属性，能否修改属性的特性，能否把属性修改为访问器属性。对于直接在对象上定义的属性，该特性默认值为true。
[[Enumerable]] 表示能否通过for-in循环返回属性。对于直接在对象上定义的属性，该特性默认值为true。
[[Get]] 在读取属性时调用的函数。默认值为undefined。
[[Set]] 在读取属性时调用的函数。默认值为undefined。 
var book = {
    _year: 2004,
    edition: 1
}
Object.defineProperty(book, "year", {
    get: function(){
        return this._year;
    },
    set: function(newValue) {
        if(newValue > 2004) {
            this._year = newValue;
            this.edition += newValue - 2004;
        }
    }
});

book.year = 2020;
alert(book.edition);   //17

_year前面的下划线是一种常用记号，用来表示只能通过对象方法访问属性。

使用访问器属性的常见方式，是设置一个属性的值，会导致其他属性发生变化。

定义多个属性

ES5定义了一个Object.defineProperties()方法，可以一次性定义多个属性，其接受两个对象参数；

  • 属性所在对象
  • 对象的属性名和对应的修改的特性
var book = {};
Object.defineProperties(book, {
    _year: {
        writable: true,
        value: 2020
    },
    edition: {
        writable: true,
        value: 1
    },
    year: {
        get: function() {
            return this._year;
        },
        set: function(newValue) {
            if(newValue > 2004) {
                this._this = newValue;
                this.edition += newValue - 2004;
            }
        }
    }
})

book.year = 2020;
alert(book.edition);   //17

读取属性的特性

ES5的Object.getOwnPropertyDescriptor()方法，可以取得给定属性的描述符，接收两个参数；

  • 属性所在对象
  • 要读取的描述符,所在的属性名称
//接着上面写
var descriptor = Object.getOwnPropertyDescriptor(book, "_year");
alert(descriptor.value);     //2020

创建对象

虽然Object构造函数或对象字面量都可以用来创建单个对象，但缺点在于，当创建很多相似对象时，会产生大量的重复代码。

工厂模式

用函数来封装以特定接口创建对象细节

function createPerson(name, age, job){
    var o = new Object();
    o.name = name;
    o.age = age;
    o.job = job;
    o.sayName = function(){
        alert(this.name);
    };
    return o;
}
var person1 = createPerson("jinhao", 20, "student");

函数内通过new关键字创建对象，添加属性和方法后，函数返回该对象。

解决了创建多个相似对象的问题，可没有解决对象识别的问题(不知道对象的类型)

构造函数模式

ECMAScript中构造函数可用来创建特定类型的对象，因此可以自定义构造函数，以此自定义对象类型的属性和方法。

function Person(name, age, job){
    this.name = name;
    this.age = age;
    this.job = job;
    this.sayName = function(){
        alert(this.name);
    };
}
var person1 = new Person("jinhao", 29, "student")  //赋值的时候，注意加引号才行

构造函数名字首字母应该要以大写字母开头。

构造函数中，没有显示地创建对象，直接将属性和方法赋给了this对象，没有return语句。

创建实例的时候必须用new关键字，该方式调用构造函数实际会经历一下4步骤；

  • 1.创建一个新对象
  • 2.将构造函数的作用域赋给新对象（因此this指向这个新对象）
  • 3.执行构造函数中的代码（为这个新对象添加属性）
  • 4.返回新对象

该方法创建的所有实例对象即是Object的实例，也是Person的实例。可通过instanceof操作符验证

alert(person1 instanceof Object);   //true
alert(person1 instanceof Person);   //true

构造函数与其他函数的唯一区别，就在于调用它们的方式不同。任何函数，只要通过new操作符来调用，那它就可作为构造函数。不通过new操作符调用，那它跟普通函数也不会有什么两样。

构造函数的主要问题，就是每个方法都要在每个实例上重新创建一遍：可以将函数定义转移到构造函数外面(全局作用域中)来解决这个问题。

function Person(name, age, job){
    this.name = name;
    this.age = age;
    this.job = job;
    this.sayName = sayName;
}
function sayName(){
    alert(this.name);
};

可是新问题又来了：①在全局作用域中定义函数实际上只能被某个对象调用，让全局作用域有点名不副实。②如果对象的方法有很多，则需要定义很多全局函数，则该自定义的应用类型就没有封装性而言了。：可通过原形模式来解决。

原型模式

① 每个函数都有一个prototype(原型)属性，这个属性是一个指针，指向本函数的原型对象。原型对象可以用来存储所有实例共享的属性和方法；原型对象也会自动获得一个constructor(构造函数)属性，这个属性也是一个指针，指向构造函数；新创建的实例，内部也有一个指针[[Prototype]]，指向构造函数的原型对象，因此实例和构造函数没有直接关系。

function Person(){
}
Person.prototype.name = "jinhao";
Person.prototype.age = 22;
Person.prototype.job = "student";
Person.prototype.sayName = function(){
    return this.name;
};

var person1 = new Person();
console.log(person1.sayName());   //jinhao

② 虽然构造函数是空函数，但我们把所有属性和方法，都存在了原型对象上。该函数的所有实例都共享这些属性和方法，读取这些属性和方法是通过查找对象属性的过程来实现的（搜索首先是从对象实例本身开始，然后搜索指针指向的原型对象）

inPrototypeOf()、Object.getPrototypeOf()

③ 实例内部包含的指针，ES5中叫做[[Prototype]],但没有标准的方式访问[[Protorype]]，但在Chrome等浏览器中每个对象都支持一个属性（__proto__）。可以通过inPrototypeOf()方法或Object.getPrototypeOf()来确定实例与原形之间的关系。

alert(Person.prototype.isPrototypeOf(person1));   //true

ES5新增了一个方法Object.getPrototypeOf()，可以返回[[Prototype]]的值

alert(Object.getPrototypeOf(person1) == Person.prototype);     //true

④ ** 可以通过对象实例访问保存在原型中的值，但不能通过对象实例重写**原型中的值。因为读取属性时，是从对象开始查找的，对象实例上添加了一个属性，这个属性就会屏蔽原型对象中保存的同名属性。

hasOwnProperty()

⑤使用hasOwnProperty()方法可以检测一个属性是存在于实例中，还是存在于原型中。只有给定属性存在于对象实例中，才会返回true。

alert(person1.hasOwnProperty("name"))     //false  虽然原型里有，但对象实例里没有

Object.keys()、Object.getOwnPropertyNames()

⑥ in操作符单独使用时，会在通过对象能够访问给定属性时返回true，无论该属性存在于实例中还是原形中。用于for-in循环中，返回的是所有能够通过对象访问的、可枚举(enumerated)的属性，既包括存在于实例中的属性，也包括存在于原型中的属性。

//in单独使用
alert("name" in person1);   //true
//for-in循环使用
for(var prop in person){
    ...
}

ES5的Object.keys()方法，可取得对象上所有可枚举的实例属性。接收一个对象作为参数，返回一个包含所有可枚举属性的字符串数组。

function Person(){
}
Person.prototype.name = "jinhao";
Person.prototype.age = 22;
Person.prototype.job = "student";
Person.prototype.sayName = function(){
    return this.name;
};

var keys = Object.keys(Person.prototype);
alert(keys);     //name,age,job,sayName

var p1 = new Person();
p1.name = "yimiao";
p1.age = 21;
var p1keys = Object.keys(p1);
alert(p1keys);   //name,age

如果想得到所有实例属性，无论它是否可枚举，都可以使用Object.getOwnPropertyNames()

var keys = Object.getOwnPropertyNames(Person.prototype);
//constructor,name,age,job,sayName

⑦由于每添加一个属性和方法就要敲一遍Person.prototype，为了减少输入，可用一个包含所有属性和方法的对象字面量来重写整个函数。

function Person(){
}
//对象字面量的方式，重写了原型对象的属性和方法
person.prototype = {
    name: "jinhao",
    age: 22,
    job: "student",
    sayName: function(){
        alert(this.name);
    }
}

但这里使用的语法，本质上完全重写了默认的prototype对象，因此constructor属性也就变成了新对象的constructor属性(指向Object函数)。可以特意将它设置回适当的值

在上面的，原型对象里，添加下面这句。
constructor: Person,

这种方式重设constructor属性会导致他的[[Enumerable]]特性被设置为true，可以通过Object.defineProperty()方法添加constructor属性，并将值设置为false，但这种方法只适用于兼容ES5的浏览器。

Object.defineProperty(person.prototype, "constructor", {
    enumerableen: false,
    value: Person
})

⑧对原型对象所做的任何修改(person.prototype.sayHi = function(){})都能立刻从实例上反映出来—即使是先创建了实例后修改原型也是如此。因为实例与原形之间的连接只不过是一个指针，而不是副本。但如果是重写整个原型对象(字面量的方式 person.prototype(){})，情况就不一样了，我们调用构造函数时会为实例添加一个指向最初原型的[[prototype]]指针，如果把原型修改为另一个对象了，就相当于切断了现有原型与最初实例对象之间的联系，最初的实例对象仍然引用的是最初的原型。

⑨原型模式的重要性不仅体现在创建自定义类型方面，就连所有原生对象的引用类型，都是采用的这种模式创建的。所有原生引用类型（Object、Array、String等等）都在其构造函数的原型上定义了方法。通过原生对象的原型，不仅可以取得所有默认方法的引用，而且也可以定义新方法。但不推荐在产品化的程序中修改原生对象的原型(如果一个实现中缺少某个方法，你添加了它。万一在另一个实现里支持这个方法，就可能会导致命名冲突，还可以意外重写原生方法)

⑩ 原生模式的缺点；

  • 省略了为构造函数传递初始化参数的环节，结果所有的实例在默认情况下都将取得相同的属性值。
  • 其共享的本质，导致了引用类型值的属性，被全部实例共享，任何一个实例修改了这个属性，其他实例也会跟着改变。

组合使用构造函数模式和原形模式

结合两种模式之长；构造函数用于定义实例属性，原型模式用于定义方法和共享的属性。结果，每个实例都会有自己的一份实例属性的副本，但同时又共享着对方的引用，最大限度地节省内存。还支持想构造函数传递参数。是目前ECMAScript中使用最广泛，认同度最高的一种创建自定义类型的方法。

//利用构造函数模式创建实例属性和方法
function Person(name, age, job){
    this.name = name;
    this.age = age;
    this.job = job;
    this.friends = ["Shelby", "Court"];
}
利用原型模式创建共享属性和方法
person.prototype = {
    constructor: Person,
    sayName: function(){
        alert(this.name);
    }
}

var person1 = new Person("jinhao", 22, "student");
var person2 = new Person("yimiao", 21, "student");

alert(person1.friends === person2.friends);   //false
alert(person1.sayName === person2.sayName);   //true

动态原型模式

动态原型模式：把所有信息都封装在构造函数中，在构造函数中初始化原型(必要情况下)。通过检查某个应该存在的方法是否有效，来决定是否需要初始化原型。

function Person(name, age, job){
    this.name = name;
    this.age = age;
    this.job = job;
    //用if语句检查应该存在的任何属性或方法，只需检查其中一个，然后借此初始化原型。
    if(typeof this.sayName != "function"){
        Person.prototype.sayName = function(){
            alert(this.name);
        };
    }
}
var person1 = new Person("jinhao", 22, "student");

寄生构造函数模式

基本思想：创建一个函数，该函数的作用仅仅是封装创建对象的代码，然后再返回新创建的对象。除了创建对象实例时使用new关键字，并且把使用的包装函数叫做构造函数外，这个模式跟工厂模式是一模一样的。

function Person(name, age, job){
    var o = new Object();
    o.name = name;
    o.age = age;
    o.job = job;
    o.sayName = function(){
        alert(this.name);
    };
    return o; 
}
var person1 = new Person("jinhao", 22, "student");
person1.sayName();   //"jinhao"

该模式，返回的对象与构造函数或者与构造函数的原型属性之间没有关系。构造函数返回的对象与在构造函数外部创建的对象没有什么不同，为此不能依赖instanceof操作符来确定对象类型。如果可以使用其他模式就不要用这种模式。

稳妥的构造函数模式

稳妥对象：没有公共属性，其方法也不引用this的对象。最适合在一些安全的环境中,或者在防止数据被其他应用程序改动时使用。

稳妥构造函数：与寄生构造函数类似但有两点不同；一是新创建对象的实例方法不引用this，二是不适用new操作符调用构造函数。 

function Person(name, age, job){
    var o = new Object();
    o.name = name;
    o.age = age;
    o.job = job;
    o.sayName = function(){
        alert(o.name);
    };
    return o; 
}
var person1 = Person("jinhao", 22, "student");
person1.sayName();    //"jinhao"

即使有代码会给这个对象添加方法或数据成员，但也不可能有别的方法访问到构造函数中的原始数据。该模式返回的对象与构造函数之间也没有什么关系，因此instanceof操作符对这种对象也没意义。

继承

许多OO语言都支持两种继承方式：接口继承和实现继承。接口继承只继承方法签名，而实现继承则继承实际的方法。ECMAScript只支持实现继承，主要依靠原型链来实现。

原型链

①基本思想是利用原型让一个引用类型继承另一个引用类型的属性和方法。将父函数的实例，赋值给子函数的原型，子函数的原型对象就会包含一个指向父函数原型的指针，层层递进，就构成了实例与原形的链条。

function SuperType(){
    this.property = true;
}
SuperType.prototype.getSuperValue = function(){
    return this.property;
};
function SubType(){
    this.subproperty = false;
}
SubType.prototype.getSubValue = function(){
    return this.subproperty;
}
//SubType继承了SuperType
SubType.prototype = new SuperType();

var instance = new SubType();
alert(instance.getSuperValue());   //true

②因为所有引用类型默认都继承了Object，这个继承也是通过原型链实现的。因此所有函数的默认原型都是Object的实例。所以原型链的末端应该是Object.prototype。

③对前面搜索机制进行补充：访问一个实例的属性时，首先会在实例中搜索该属性，然后再实例的原型中进行搜索，接着如果通过原型链实现继承的情况下，搜索过程会沿着原型链继续向上，知道搜索到原型链的末端。

④有两种方式来确定原型与实例之间的关系；

  • 使用instanceof操作符——测试实例与原型链中出现过的构造函数，结果都会返回true

    //由于原型链的关系，可以说instance是Object、SuperType和SubType的实例
alert(instance instanceof Object);      //true
alert(instance instanceof SuperType);   //true
alert(instance instanceof SubType);     //true

  • 使用inPrototypeOf()只要原型链中出现过的原型，都可以说是该原型链所派生的实例的原型。因此也会返回true

    alert(Object.prototype.isPrototypeOf(instance));       //true
alert(SuperType.prototype.isPrototypeOf(instance));    //true
alert(SubType.prototype.isPrototypeOf(instance));      //true

⑤子类型需要覆盖超类型中的某个方法，或者需要添加超类型中不存在的某个方法，可以在子函数的原型中添加相应的方法实现“添加”，“覆盖”。，给子函数的原型添加方法的代码一定要放在建立继承的语句之后。且不能使用字面量创建原型的方法，否则会重写原型链。

//......
//SubType继承了SuperType
SubType.prototype = new SuperType();

//给原型对象添加或修改某个方法
SubType.prototype.getSubValue = fnction(){
    return this.subproperty;
}

//不能使用字面量的方式，会导致继承的代码无效。以下是错误的。
(错误)  SubType.prototype = {
            getSubValue: function(){
                return this.subproperty;
            }
        }

⑥原型链存在的问题：①其共享的特性，就会导致引用类型值的原型，使用时与预期不符。②创建子类型实例的时候，虽然能访问到超类的属性和方法，但不能向超类型的构造函数传递参数。因此很少单独使用原型链。

借用构造函数

①借用构造函数的技术有时也叫做“伪类对象”或“经典继承”。基本思想是：在子类型构造函数的内部调用超类型的构造函数。

function SuperType(){
    this.colors = ["red", "blue", "green"];
}
function SubType(){
    //继承了SuperType
    SuperType.call(this)
}
var instance1 = new SubType();
instance1.colors.push("black");
alert(instance1.colors);     //"red,blue,green,black"

var instance2 = new SubType();
alert(instance2.colors);     //"red,blue,green"

②通过使用call()方法或apply()方法，可以在新创建SubType实例的环境下调用SuperType构造函数。结果，SubType每个实例都会有一份SuperType属性和方法的副本。通过该方法还可以向超类型构造函数传递参数；

function SuperType(name){
    this.name = name;
}
function SubType(){
    //继承了SuperType,同时传递参数
    SuperType.call(this, "jinhao");
    //实例属性
    this.age = 22;
}

var instance = new SubType();
alert(instance.name);      //"jinhao"
alert(instance.age);       //22;

③借用构造函数存在的问题：①无法避免构造函数存在的问题，即方法都在构造函数中定义，函数复用就无从谈起了。②在超类型的原型中定义方法，对子类型而言是不可见的，结果，所有类型都只能使用构造函数的方式。因此很少单独使用原型链。

组合继承

组合继承有时也叫伪经典继承，指的是将原型链和借用构造函数的技术组合到一块,融合他们的优点。思路是：使用原型链实现对原型属性和方法的继承，通过借用构造函数来实现对实例属性的继承。

function SuperType(name){
    this.name = name;
    this.colors = ["red", "blue", "green"];
}
SuperType.prototype.sayName = function(){
    alert(this.name)
}

function SubType(name, age){
    //继承属性
    SuperType.call(this, name);
    this.age = age;
}
//继承方法
SubType.prototype = new SuperType();
SubType.prototype.constructor = SubType;
SubType.prototype.sayAge = function(){
    alert(this.sayAge);
};

var instance1 = new SubType("jinhao", 22);
instance1.colors.push("black");
alert(instance1.colors);     //"red,blue,green,black"
instance1.sayName();         //"jinhao"
instance1.sayAge();          //22

该模式是javascript中最常用的继承模式。而且instanceof和isPrototypeOf()也能够用于识别基于组合继承创建的对象。

原型式继承

借助原型可以基于已有的对象创建新对象，同时还不必因此创建自定义类型。

function object(o){
    function F(){};
    F.prototype = o;
    return new F();
}

在objcet()函数内部，先创建了一个临时性的构造函数，然后将传入的对象作为这个构造函数的原型，最后返回这个临时类型的一个新实例。这种原型式继承，要求必须要有一个对象可以作为另一个对象的基础，然后再根据具体需求对得到的对象加以修改即可。本质上讲，object()对传入的对象进行了一次浅复制（对引用类型属性来说，只拷贝了引用，而不是对象本身）

//接着上面
var person = {
    name: "jinhao",
    friends: ["shelby", "count", "van"]
}

var anotherPerson = object(person);
anotherperson.name = "yimiao";      //根据需求得到的对象加以修改
anotherperson.friends.push("rob");

var yetAnotherPerson = object(person);
yetAnotherPerson.name = "linda";
yetAnotherPerson.friends.push("barbie");

alert(person.friends);    //"shelby、count、van、rob、barbie"

ES5新增Object.create()方法规范化了原型化继承，接收两个参数：一个用作新对象原型的对象、一个为新对象定义额外属性的对象(可选)。

var person = {
    name: "jinhao",
    friends: ["shelby", "count", "van"]
}

var anotherPerson = Object.create(person);
anotherPerson.name = "yimiao";      //根据需求得到的对象加以修改
anotherPerson.friends.push("rob");

var yetAnotherPerson = Object.create(person, {
    name: {
        value: "Greg"
    }
});

只想让一个对象与另一个对象保持类似的情况下，原型式继承是完全可以胜任的。但包含引用类型值的属性始终都会共享相应的值，就像使用原型模式一样。

寄生式继承

寄生式继承与寄生构造函数和工厂函数模式类似，即创建一个仅用于封装继承过程的函数，该函数在内部已某种方式来增强对象，最后再像真地是它做了所有工作一样地返回对象。

function createAnother(original){
    var clone = object(original);  //通过调用函数创建一个新对象
    clone.sayHi = function(){      //以某种方式来增强对象
        alert("hi");
    };
    return clone;     //返回这个对象
}

object()函数不是必须的，任何能够返回新对象的函数都适用于此模式。

//接着上面
var person = {
    name: "jinhao",
    friends: ["shelby", "count", "van"]
}
var anotherPerson = createAnother(person);
anotherPerson.sayHi();     //"hi"

新对象anotherPerson，不仅具有person的所有属性和方法，而且还有自己的sayHi()方法。

寄生组合式继承

组合继承最大的问题就是，无论什么情况下，都会调用两次超类型的构造函数；

function SuperType(name){
    this.name = name;
    this.colors = ["red", "blue", "green"];
}
SuperType.prototype.sayName = function(){
    alert(this.name)
}

function SubType(name, age){
    SuperType.call(this, name);           //第二次调用SuperType()
    this.age = age;
}
SubType.prototype = new SuperType();      //第一次调用SuperType()
SubType.prototype.constructor = SubType;
SubType.prototype.sayAge = function(){
    alert(this.sayAge);
};

第一次调用SuperType构造函数时，SubType.prototype会得到两个属性：name，colors。

第二次调用SuperType构造函数时，又会在SubType的实例上创建了两个属性：name，colors，实例属性会屏蔽原型中的两个同名属性。

寄生组合式继承，的思路是：不必为了指定子类型的原型而调用超类型的构造函数，我们所需要的只是超类型原型的一个副本而已。从而减少一次调用SuperType函数， 避免了在SubType.prototype上创建不必要，多余的属性。

function inheritPrototype(subType, superType){
    var prototype = object(subType.prototype);   //创建对象
    prototype.constructor = surType;             //增强对象
    subType.prototype = prototype;               //指定对象
}

第一步是创建超类型原型的一个副本，第二步为创建的副本添加constructor属性，弥补因重写原型而失去默认的constructor属性属性。最后将创建的对象（即副本）赋值给子类型的原型。

然后我们就可以调用inheritPrototype(subType, superType)去替换前面例子中为子类型原型赋值的语句了。

function SuperType(name){
    this.name = name;
    this.colors = ["red", "blue", "green"];
}
SuperType.prototype.sayName = function(){
    alert(this.name)
}

function SubType(name, age){
    SuperType.call(this, name);           //只调用了SuperType()一次
    this.age = age;
}
inheritPrototype(subType, superType);

SubType.prototype.sayAge = function(){
    alert(this.sayAge);
};
